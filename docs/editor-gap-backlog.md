# 编辑器能力缺口留档

日期：2026-06-16

本文记录当前项目在“AI HTML Slides 到人类可编辑 UI”方向上仍缺少的能力，用作后续计划书、任务拆分和验收标准的来源。本文不是单轮迭代计划，优先记录问题空间。

## 1. 当前已具备能力

项目当前已经具备以下基础能力：

- 多页 demo deck。
- 左侧幻灯片缩略图。
- 中间固定比例画布。
- 右侧属性面板。
- 图层面板。
- 单选、多选、框选。
- 多元素移动和缩放。
- 元素拖动、缩放、旋转。
- flat `groupId` 组合 / 取消组合。
- 文本、图片、形状、HTML 元素基础渲染。
- 文本内容、字体、字号、颜色、行高、字距等基础编辑。
- 图片替换、适应方式调整。
- 形状填充、边框、阴影。
- 页面背景纯色、渐变、图片背景。
- 元素锁定、隐藏、复制、删除、快捷键。
- 导入受约束 HTML。
- 保存 JSON。
- 导出单文件 HTML。
- 基础图片导入。

当前短板已经从“能不能编辑元素”转向“能不能稳定完成一整套演示文稿的创建、维护、保存、导出和复用”。

## 2. 项目级能力缺口

目前更像单次编辑器，还不像完整的本地项目工具。

缺少：

- 打开本地 `deck.json`。
- 自动保存草稿。
- 恢复草稿提示。
- 未保存修改提示。
- 项目名称、作者、创建时间、修改时间。
- 最近打开项目。
- 项目保存状态：已保存 / 未保存 / 有草稿。
- 项目版本迁移机制，例如 `0.1 -> 0.2`。
- 错误恢复机制，例如导入失败后保留当前 deck。

优先级判断：

- 高优先级：打开 JSON、自动草稿、未保存提示。
- 中优先级：项目元信息、最近项目。
- 后置：版本迁移、复杂恢复策略。

## 3. 页面管理缺口

当前可以切换已有页面，但不够支持从零组织一套 slides。

缺少：

- 新建页面。
- 复制页面。
- 删除页面。
- 页面重命名。
- 页面排序。
- 页面尺寸设置：16:9、4:3、自定义。
- 页面备注。
- 页面模板 / 版式选择。
- 批量选择多个页面。
- 批量设置页面背景。

优先级判断：

- 高优先级：新建、复制、删除、重命名、排序。
- 中优先级：备注、页面背景批量设置。
- 后置：多尺寸、批量页面操作。

## 4. 元素创建缺口

当前更适合编辑导入内容，从空白页面开始搭建内容的能力不足。

缺少：

- 插入标题。
- 插入正文。
- 插入图片占位。
- 插入矩形、圆形、线条、箭头。
- 插入图标。
- 插入按钮 / 标签。
- 插入卡片。
- 插入分割线。
- 插入表格。
- 插入图表占位。
- 插入代码块。
- 插入引用块。
- 插入流程图节点。

优先级判断：

- 高优先级：标题、正文、图片占位、基础形状、线条。
- 中优先级：卡片、标签、分割线。
- 后置：表格、图表、代码块、流程图。

## 5. 图层与选择逻辑缺口

当前图层面板已经可以查看、选择、隐藏、锁定元素，但还不完整。

缺少：

- 图层重命名。
- 图层拖拽排序。
- 上移一层 / 下移一层 / 置顶 / 置底。
- 多图层选择。
- 图层搜索。
- 组折叠 / 展开。
- 锁定组。
- 隐藏组。
- 图层面板右键菜单。
- 被遮挡元素的选择列表弹窗。
- 画布右键菜单。

优先级判断：

- 高优先级：层级调整、图层重命名、右键菜单。
- 中优先级：组折叠、多图层选择、选择列表弹窗。
- 后置：图层搜索、复杂组管理。

## 6. 对齐与排版缺口

当前能拖动和缩放，但缺少高频排版辅助。

缺少：

- 左对齐、水平居中、右对齐。
- 顶对齐、垂直居中、底对齐。
- 多元素水平分布。
- 多元素垂直分布。
- 吸附到画布中心线。
- 吸附到画布边缘。
- 吸附到其他元素边缘。
- 吸附到其他元素中心。
- 参考线显示。
- 网格显示。
- 标尺。
- 元素间距提示。
- 一键等宽 / 等高。
- 一键匹配大小。

优先级判断：

- 高优先级：对齐、分布、中心线吸附。
- 中优先级：参考线、边缘吸附、间距提示。
- 后置：标尺、复杂网格系统。

## 7. 文本能力缺口

当前文本框整体样式已经可编辑，但不支持细粒度富文本。

缺少：

- 文本框内局部加粗。
- 局部颜色。
- 局部字号。
- 项目符号。
- 编号列表。
- 段落间距。
- 首行缩进。
- 文本自动适应高度。
- 文本自动缩小以适配框。
- 字体收藏。
- 字体缺失检查。
- 字体导出策略。
- 字体嵌入策略。

优先级判断：

- 高优先级：文本溢出检查、自动适配高度、字体收藏。
- 中优先级：项目符号、编号列表、段落间距。
- 后置：局部富文本、字体嵌入。

## 8. 媒体能力缺口

当前已经可以导入图片元素，但媒体系统还没有形成资源库。

缺少：

- 媒体资源库。
- 图片复用到多页。
- 替换图片但保留位置、尺寸和样式。
- 图片裁切。
- 图片裁切位置调整。
- 图片蒙版：圆形、圆角矩形、自定义形状。
- 图片压缩。
- 图片尺寸信息。
- 外链图片转 Data URL。
- SVG 导入策略。
- GIF 支持策略。
- 视频导入。
- 音频导入。
- iframe 嵌入，例如网页、地图、视频链接。
- 媒体导出前检查。

优先级判断：

- 高优先级：资源库、图片复用、替换图片、裁切位置。
- 中优先级：图片压缩、尺寸信息、外链转 Data URL。
- 后置：视频、音频、iframe、复杂蒙版。

## 9. 样式系统缺口

当前元素样式分散在各自 `style` 字段中，缺少全局风格抽象。

缺少：

- 主题色。
- 全局字体。
- 标题样式。
- 正文样式。
- 卡片样式。
- 默认阴影。
- 默认圆角。
- 一键套用主题。
- 当前页面色板。
- 最近使用颜色。
- 取色器。
- 样式复制 / 粘贴。
- 清除样式。

优先级判断：

- 高优先级：最近使用颜色、样式复制 / 粘贴、全局字体。
- 中优先级：主题色、标题/正文样式。
- 后置：复杂主题系统。

## 10. 模板与版式缺口

要接近可画、PPT 或 Figma 类工具，需要逐步支持模板和版式。

缺少：

- 单页模板。
- 封面模板。
- 目录页模板。
- 对比页模板。
- 时间线模板。
- 图片墙模板。
- 数据页模板。
- 结束页模板。
- 卡片模板。
- 用户自定义模板。
- 从当前页面保存为模板。

优先级判断：

- 高优先级：少量内置版式 preset。
- 中优先级：从当前页面保存为模板。
- 后置：模板中心、模板分类、模板预览市场。

## 11. 导入导出缺口

当前支持导入受约束 HTML、保存 JSON、导出 HTML，但缺少质量控制和更多格式。

缺少：

- 打开 JSON。
- 导出前检查。
- 文本溢出检查汇总。
- 图片缺失检查。
- 外链资源检查。
- 字体缺失检查。
- 隐藏元素提示。
- HTML 重新导入一致性测试。
- 导出为 PNG。
- 导出为 PDF。
- 导出为 PPTX。
- 导出 zip 包。
- 导出时选择是否内联资源。

优先级判断：

- 高优先级：打开 JSON、导出前检查、HTML 重新导入一致性。
- 中优先级：PNG/PDF 导出。
- 后置：PPTX、zip、复杂资源策略。

## 12. 演示播放缺口

当前是编辑器，不是播放器。HTML slides 项目最终需要具备演示能力。

缺少：

- 播放模式。
- 全屏演示。
- 键盘翻页。
- 当前页码。
- 演讲者备注。
- 计时器。
- 激光笔。
- 导出演示 HTML。
- 动画预览。
- 交互元素预览。

优先级判断：

- 高优先级：基础播放模式、全屏、键盘翻页。
- 中优先级：备注、计时器、页码。
- 后置：激光笔、复杂交互预览。

## 13. 动画与交互缺口

当前元素动画和演示交互基本没有。

缺少：

- 元素入场动画。
- 元素退场动画。
- 强调动画。
- 页面切换动画。
- 动画顺序。
- 动画延迟。
- 动画时长。
- 点击触发。
- 自动播放。
- 交互按钮。
- 展开 / 折叠区域。
- hover 效果。

优先级判断：

- 高优先级：元素基础入场动画字段和导出渲染。
- 中优先级：页面切换动画、动画预览。
- 后置：时间轴编辑器、复杂触发器。

## 14. 工程结构缺口

功能增加后，当前 store 和组件会继续变大，需要提前控制边界。

缺少：

- 独立 command 层。
- 独立 asset manager。
- 独立 page operations。
- 独立 layer operations。
- 统一右键菜单 action registry。
- schema migration。
- E2E 测试。
- 可视化回归截图。
- 大 deck 压测。
- 错误边界。
- 空状态和异常状态统一处理。

优先级判断：

- 高优先级：command 层、page operations、layer operations。
- 中优先级：asset manager、E2E 测试、错误边界。
- 后置：视觉回归、大 deck 压测。

## 15. 桌面化前置缺口

最终目标是个人可部署并打包为 EXE，但当前还缺桌面化前置能力。

缺少：

- 本地文件读写模型。
- 最近文件列表。
- 自动保存到本地文件。
- 资源文件夹管理。
- 离线字体策略。
- 离线图片策略。
- 桌面应用菜单。
- Electron / Tauri 技术选型。
- 打包体积评估。
- 自动更新策略。

优先级判断：

- 高优先级：先让 Web 端的数据结构适合本地项目文件。
- 中优先级：研究 Tauri / Electron。
- 后置：真正打包 EXE、自动更新。

## 16. 推荐后续收敛方向

后续不要一次性覆盖所有缺口，建议按以下路线收敛：

1. 先收口编辑命令层，避免功能继续分散到 toolbar、keyboard、layer、canvas 中。
2. 完善右键菜单、图层层级、页面管理，让已有编辑能力更像 PPT/Figma。
3. 引入媒体资源库，让图片从“一次性导入元素”升级为“项目资产”。
4. 补本地保存和恢复，保证个人长期使用不丢内容。
5. 再扩展对齐吸附、模板、动画、播放模式、桌面打包。

下一轮计划建议优先做：

- command 层收口。
- 右键菜单。
- 图层层级管理。
- 页面管理。
- 媒体资源库第一版。
