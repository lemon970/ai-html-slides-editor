# 第八轮迭代计划：可靠性收口、导出安全与媒体资源地基

日期：2026-06-18  
分支基准：`codex/practical-usability-task-1-4`

## 一、定位

第七轮已加入 Snap、代码块、PNG/PDF 导出、Source HTML bridge 和 UI 重排。本轮不继续堆复杂新能力，而是先处理第七轮后暴露的风险：

- 验证链路必须可重复执行，`pnpm test` 和 `pnpm build` 不能被安装策略阻断。
- code/html 元素不能在编辑器或导出文件中执行不可信脚本。
- 草稿不能为了压缩体积静默破坏图片地址。
- Source HTML 导入失败时应回落到预览模式，而不是直接中断。
- 图片资源需要进入可复用的媒体库，为后续视频、字体、动画和桌面版资源目录打基础。

## 二、本轮范围

### A. 验证与仓库卫生

- 明确允许当前依赖所需的 build script。
- 将本地 `.claude/` 和 `对话历史.jsonl` 排除在业务提交之外。
- 保持验证命令为：
  - `pnpm test`
  - `pnpm build`

### B. 安全渲染

- code 元素中的 `html` 字段按源码文本处理。
- Shiki 失败时回退到 escape 后的 `<pre><code>`。
- 普通 html 元素默认按不可信 HTML 处理，导出和编辑器渲染前净化。
- 编辑器渲染、HTML 导出、PNG/PDF 导出共享同一套安全 HTML 输出。

### C. 草稿持久化

- 大草稿不再把图片 `src` 写成 `"[omitted]"`。
- 图片被省略时写入明确状态，并在恢复草稿时提示“部分图片未保存”。
- 读取草稿时使用 schema 校验，坏草稿直接忽略。

### D. Source HTML 容错

- `__DECK_JSON__` 优先识别为完整 JSON deck。
- 只有同时存在 `data-deck`、`data-slide`、`data-element` 时才按受约束 HTML 解析。
- 解析失败时自动进入 Source HTML 预览模式，并显示失败原因。
- iframe 沙箱默认移除 `allow-same-origin`，只保留脚本预览能力。

### E. 媒体资源库第一版

- `Deck` 增加 `assets?: DeckAsset[]`。
- 图片元素增加 `assetId?: string`。
- 新导入图片同时写入媒体库并插入当前页。
- Media 面板支持插入已有图片、替换当前选中图片。
- 替换图片时保留原元素位置、尺寸、旋转、层级和样式。

## 三、不做

- 不接 AI API。
- 不做 PPTX。
- 不做 EXE 打包。
- 不做视频/音频/字体嵌入。
- 不做复杂动画时间轴。
- 不做云端素材库。

## 四、验收标准

- `pnpm test` 通过。
- `pnpm build` 通过。
- 代码块内写入 `<script>alert(1)</script>` 时，只作为文本显示。
- 普通 HTML 元素导出前会移除脚本和事件属性。
- 大图片草稿恢复时能提示图片数据被省略，不会写入 `"[omitted]"`。
- 含 `data-deck` 但非受约束结构的 HTML 能回落到 Source HTML 模式。
- 图片导入后出现在媒体库，可跨页插入，可替换选中图片。
- PNG/PDF 导出遇到缺失图片或空 blob 时给出错误，不静默失败。

## 五、后续方向

第九轮可以进入“媒体能力扩展”：

- 视频/音频元素。
- 字体导入和字体资产。
- 图片裁切、蒙版和形状填充。
- 基础动画字段和播放时序。
- 桌面化资源目录方案预研。
